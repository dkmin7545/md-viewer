# 모바일 MD 파일 뷰어 — 요건서

> 작성일: 2026-04-07  
> 목적: Cowork에서 생성된 마크다운 파일을 모바일 환경에서 편리하게 열람

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 앱 유형 | 모바일 웹앱 (PWA 지향) — React + Vite 기반 |
| 대상 플랫폼 | iOS Safari / Android Chrome |
| 파일 소스 | 로컬 파일 업로드 또는 Obsidian Sync 폴더 연동 |
| 핵심 가치 | 노안 친화적 가독성 · 빠른 탐색 · 오프라인 지원 |

---

## 2. 핵심 기능 요건

### 2-1. 파일 열기

| 기능 | 상세 |
|------|------|
| 로컬 업로드 | 파일 선택 버튼으로 `.md` 파일 1개 이상 열기 |
| 폴더 업로드 | 폴더 선택 시 하위 `.md` 파일 일괄 인식 |
| 드래그 앤 드롭 | 데스크톱 브라우저에서도 지원 |
| 최근 파일 목록 | localStorage 기반 최근 열람 파일 5개 표시 |

### 2-2. 마크다운 렌더링

| 요소 | 지원 여부 |
|------|----------|
| 헤딩 H1–H6 | ✅ |
| 굵게, 기울임, 취소선 | ✅ |
| 코드 블록 (syntax highlight) | ✅ |
| 인라인 코드 | ✅ |
| 표 (table) | ✅ |
| 체크박스 (- [ ] / - [x]) | ✅ |
| 인용문 (blockquote) | ✅ |
| 이미지 (로컬 경로 제외, URL만) | ✅ |
| 내부 링크 (`[[파일명]]`) | 🔲 향후 검토 |
| Mermaid 다이어그램 | 🔲 향후 검토 |

> 라이브러리 권장: `marked` + `highlight.js` (또는 `react-markdown` + `remark-gfm`)

### 2-3. 글씨 크기 조절 ⭐ (핵심)

- **기본 글자 크기: 18px** (일반 앱 대비 크게 설정)
- 화면 하단 상시 노출되는 **크기 조절 바** (슬라이더)
  - 범위: 14px ~ 28px
  - 단계: 2px 간격
  - 현재 크기 숫자 표시
- **빠른 프리셋 버튼** 3단계: 보통(18) / 크게(22) / 매우크게(26)
- 선택한 크기는 localStorage에 저장 → 앱 재시작 시 유지
- 줄 간격(line-height)은 글자 크기에 비례 자동 조정 (1.7 고정)

### 2-4. 파일 탐색

- **사이드 패널** (왼쪽 스와이프 또는 햄버거 메뉴)
  - 업로드된 파일 목록 표시
  - 파일명 검색 필터
  - 탭 형태로 여러 파일 열람 (최대 10개)
- **목차 패널** (오른쪽 스와이프 또는 헤딩 아이콘)
  - 현재 파일의 H1–H3 목차 자동 생성
  - 탭 눌러 해당 섹션으로 스크롤

### 2-5. 읽기 편의 기능

| 기능 | 상세 |
|------|------|
| 다크/라이트 모드 | 시스템 설정 자동 감지 + 수동 토글 |
| 줄 너비 제한 | 최대 680px (모바일 가독성 최적화) |
| 스크롤 진행 표시 | 상단 얇은 프로그레스 바 |
| 헤딩 앵커 | H1–H2 탭 시 URL 해시 업데이트 |
| 전체화면 모드 | 상단/하단 UI 숨기고 읽기에 집중 |

## 3. UI 레이아웃

```
┌─────────────────────────────┐
│ ☰  파일명.md         🌙  Aa  │  ← 상단 바
├─────────────────────────────┤
│                             │
│  (렌더링된 마크다운 본문)      │
│                             │
│                             │
├─────────────────────────────┤
│  [보통]  [크게]  [매우크게]   │
│  ─────●──────────────  18px │  ← 하단 크기 조절
└─────────────────────────────┘
```

- 상단 바: 파일 목록 열기(☰), 파일명, 다크모드(🌙), 글자크기 패널 토글(Aa)
- 하단 바: 항상 노출, 글자 크기 슬라이더 + 프리셋

## 4. 기술 스택 (권장)

| 영역 | 선택 |
|------|------|
| 프레임워크 | React 18 + TypeScript |
| 빌드 | Vite |
| 마크다운 파서 | `react-markdown` + `remark-gfm` |
| 코드 하이라이트 | `react-syntax-highlighter` |
| 스타일 | Tailwind CSS v3 |
| 상태 관리 | Zustand (경량) |
| PWA | `vite-plugin-pwa` |

## 6. 개발 단계

| Phase | 작업 | 예상 소요 |
|-------|------|----------|
| **P0** | 프로젝트 셋업 + 파일 업로드 + 기본 렌더링 | 1–2시간 |
| **P1** | 글자 크기 바 + 다크모드 + 파일 탐색 패널 | 2–3시간 |
| **P2** | 목차 패널 + 진행 표시 + 탭 열람 | 2–3시간 |
| **P3** | PWA 설정 + 아이콘 + 홈 화면 추가 | 1시간 |

---

## 7. 시작 명령어

```bash
# 프로젝트 생성
npm create vite@latest md-viewer -- --template react-ts
cd md-viewer

# 의존성 설치
npm install react-markdown remark-gfm react-syntax-highlighter
npm install zustand
npm install -D tailwindcss postcss autoprefixer
npm install -D vite-plugin-pwa

# 개발 서버
npm run dev
```

## 8. 비기능 요건

- 오프라인 지원: PWA 서비스워커로 앱 자체 캐싱
- 성능: 1MB 이하 마크다운 파일 100ms 이내 렌더링
- 접근성: 터치 타깃 최소 44×44px 준수
- 브라우저: iOS 16+ / Android Chrome 110+
